# Governance Documentation
# Longitudinal AI Governance Whitepaper

## Authority Order
Human Authority → Standards → Governance → Licensing → Automation

## Governance Workflow
1. Changes proposed via pull request
2. Human authority review required
3. Approval required before merge
4. Checksums regenerated if artifacts change
5. Release tagged explicitly

## Human-Only Actions
- Canonical approval
- Version tagging
- License interpretation
- Governance modification

## Enforcement
- CODEOWNERS
- Branch protection
- Checksums
